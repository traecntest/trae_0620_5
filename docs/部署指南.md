## 三、快速开始（开发环境）

### 3.1 克隆项目

```bash
git clone <repository-url>
cd trae_0620_5
```

### 3.2 安装 PostgreSQL

**Windows:**
1. 下载 PostgreSQL 14+ 安装包：https://www.postgresql.org/download/windows/
2. 安装时设置密码为 `password`，端口 `5432`
3. 创建数据库 `elderly_canteen`

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install postgresql postgresql-contrib -y
sudo -u postgres psql
postgres=# CREATE DATABASE elderly_canteen;
postgres=# ALTER USER postgres PASSWORD 'password';
postgres=# \q
```

### 3.3 配置环境变量

编辑 `backend/.env` 文件：

```bash
DATABASE_URL=postgresql://postgres:password@localhost:5432/elderly_canteen
SECRET_KEY=your-secret-key-here-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
SMS_TEST_CODE=123456
SMS_EXPIRE_MINUTES=5
VOICE_API_KEY=your-voice-api-key
VOICE_SECRET_KEY=your-voice-secret-key
VOICE_APP_ID=your-voice-app-id
WEATHER_API_KEY=your-weather-api-key
HOST=0.0.0.0
PORT=8000
```

### 3.4 初始化数据库

**Windows:**
```cmd
cd backend
init_db.bat
```

**Linux:**
```bash
cd backend
chmod +x init_db.sh
./init_db.sh
```

初始化将创建：
- 1名管理员（13800000000 / admin123）
- 5名老人用户（13800000001 ~ 13800000005）
- 2名亲属用户（13900000001 ~ 13900000002）
- 20道示例菜品（早餐6道、午餐8道、晚餐6道）
- 90天历史订单数据（用于模型训练）

### 3.5 启动服务

**Windows:**
```cmd
cd backend
start.bat
```

**Linux:**
```bash
cd backend
chmod +x start.sh
./start.sh
```

### 3.6 访问系统

| 模块 | 地址 | 说明 |

## 四、生产环境部署（Linux）

### 4.1 系统准备

```bash
# 更新系统
sudo apt-get update && sudo apt-get upgrade -y

# 安装必要软件
sudo apt-get install -y python3 python3-pip python3-venv nginx postgresql postgresql-contrib
```

### 4.2 创建虚拟环境

```bash
cd /opt
sudo git clone <repository-url> elderly_canteen
cd elderly_canteen/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 4.3 配置 Systemd 服务

创建 `/etc/systemd/system/elderly-canteen.service`：

```ini
[Unit]
Description=社区老年食堂智能点餐系统
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/elderly_canteen/backend
Environment="PATH=/opt/elderly_canteen/backend/venv/bin"
ExecStart=/opt/elderly_canteen/backend/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable elderly-canteen
sudo systemctl start elderly-canteen
sudo systemctl status elderly-canteen
```

### 4.4 配置 Nginx 反向代理

创建 `/etc/nginx/sites-available/elderly-canteen`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 10M;

    # 静态文件
    location /static/ {
        alias /opt/elderly_canteen/frontend/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # API 代理
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/elderly-canteen /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4.5 配置 HTTPS（推荐）

使用 Let's Encrypt 免费证书：

```bash
sudo apt-get install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com
```

## 六、智能预测模块

### 6.1 模型训练

系统首次启动时会自动训练预测模型，也可以手动触发：

```bash
cd backend
python3 -c "from app.ml.prediction_service import PredictionService; ps = PredictionService(); ps.train_model(); print('训练完成')"
```

### 6.2 定时任务

系统内置两个定时任务：
1. **每日凌晨 2:00** - 生成次日备餐预测
2. **每周日凌晨 1:00** - 重新训练预测模型

可以在 `backend/app/scheduler.py` 中调整执行时间。

### 6.3 预测算法说明

**特征变量（15个）：**
- 时间特征：星期几、是否周末、月份、季节、是否节假日
- 天气特征：温度、天气状况（晴/雨/雪）
- 历史特征：昨日销量、上周同日销量、上月同日销量、近7天均值
- 其他特征：是否有社区活动、菜品价格、是否新菜品

**模型选择：**
- 梯度提升回归（Gradient Boosting Regression）
- 优势：处理非线性关系、抗过拟合能力强、特征重要性可解释

**评估指标：**
- MAE（平均绝对误差）
- RMSE（均方根误差）
- MAPE（平均绝对百分比误差）

## 八、常见问题

### Q1: 数据库连接失败
**A:** 检查 PostgreSQL 服务是否启动，确认 `.env` 中的数据库连接信息正确。

### Q2: 语音识别无法使用
**A:** 未配置语音API密钥时，系统会使用Mock识别模式。生产环境请配置百度AI语音API。

### Q3: 预测准确率低
**A:** 系统需要至少30天的历史数据才能获得较好的预测效果。可以通过 `init_db.py` 生成测试数据。

### Q4: 定时任务不执行
**A:** 检查 APScheduler 是否正常启动，查看应用日志。生产环境建议使用系统级定时任务（如 cron）作为备份。

### Q5: 老人端界面显示异常
**A:** 建议使用 Chrome、Safari 等现代浏览器，确保 JavaScript 已启用。
